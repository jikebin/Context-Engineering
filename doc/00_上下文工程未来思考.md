# 对于上下文工程的未来思考

## 基于code 环境自主构建上下文工程Agent

### 训练阶段

- 工具：给模型提供一个 Code Interpreter 工具
- 提示词： 让模型意识到自己上下文是有限的，让模型通过自主构建上下文工程需要的环境，以及自主进行压缩、卸载、记录、检索等逻辑来形成自主的能力构建。从而形成自己真正的“记忆”模块。

### 可能思考的形式

- 首先构建一个token统计模块，可以让模型来自己统计token占用和token剩余。
- 然后构建外部的文档保存模块，来进行检索和存储。
- 记忆分层：有TODO list(正在做)、Plan(计划做)、Done(已完成)、History(历史记录)
