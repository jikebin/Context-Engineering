# 上下文工程的操作系统思维

- LLM = CPU：强大的推理计算核心，但自身无状态、不存储信息。
- 上下文窗口 = RAM：高速的“工作记忆”，单容量有限且内容易失。
- 上下文工程 = OS：扮演内存管理器，负责从“硬盘”向 “RAM“ 高效调度关键信息。

## OS需要管理的三大“数据类型”

### 指令（Instructions）

Agent的行动纲领（Prompts,工具定义，思维链）

### 知识（Knowledge）

决策的事实依据（检索的文档，用户画像）

### 工具反馈（Tool Feedback）

与世界交互的观察结果（API返回，文件内容）

## 上下文工程的四大核心操作（WSCI 框架）

- 一个并行的 “工具箱”，而非 线性的“流水线”

### 写入（Write）——积累经验

将 Agent 的观察、思考和记忆存入“草稿纸” 或 “长期记忆”，是信息产生的源头。

### 选择（Select）——专注重点

每次调用LLM前的核心动作，从所有信息源中挑选最相关的内容，是信噪比管理的关键。

### 压缩（Compress）——长期工作

“选择”的辅助手段，当信息过长时按需触发，通过总结或裁剪来节省空间。

### 隔离（Isolate）——安全扩展

顶层架构决策，为不同子任务或子Agent划分独立上下文，避免信息干扰。
