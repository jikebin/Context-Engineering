# 动态上下文的两种范式

- AI信息检索的进化：从“被动投喂”到“主动探索”。

## 推理前检索（RAG）

- 被动的信息投喂
- 图书管理员模式

### 工作流

单向线性：提问-〉检索-〉增强上下文-〉生成答案。

### 优点

- 速度相对较快
- 架构成熟、稳定
- 适合静态知识库

### 缺点

- 信息可能过时
- 易引入检索噪声
- 无法处理探索性任务

## 即时检索（Agentic Search）

- 主动的知识探索
- 侦探/研究员模式

### 工作流

循环动态：思考-〉调用工具-〉观察-〉再思考...

### 优点

- 信息永远最新（实时）
- 高信噪比，上下文精准
- 能处理复杂、多步骤探索任务

### 缺点

- 速度较慢、延迟高
- 系统设计复杂
- Agent可能“迷路”或低效探索。

## Agentic Search 的智能之源

### 利用元数据

不仅理解内容，更能理解**文件名、目录结构、API状态吗**等“启发式信号”进行推理。

### 渐进式披露

通过主动探索，**一层层地揭示**上下文，避免信息过载，始终聚焦于核心任务。

### 最终答案：混合策略

**预加载**高阶引导信息（RAG思维）+赋予 Agent 即时探索工具（Agentic思维） = 平衡速度、灵活性与智能度的最佳实践。
